Reports a class member that reuses a supertype's type-parameter name (for example T from implements Holder<Object>) where that parameter is not declared in the current scope.

The compiler does not bind such a name — it must be written as the concrete bound type. The quick-fix replaces the bare parameter (e.g. T) with the type the supertype clause binds it to (e.g. Object).

A class that actually declares the parameter (class Box<T> implements Holder<T>) or a generic method (<T> …) is never flagged.
