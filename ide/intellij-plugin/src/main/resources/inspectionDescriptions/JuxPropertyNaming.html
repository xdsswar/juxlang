Reports observable properties whose name does not start with an uppercase letter (W0974, §P.1.1).

PascalCase property names are the visual signal that a name is a { get; set; } property rather than a plain field. This is a convention, never an error — camelCase properties compile fine. The quick-fix renames the property and all resolving usages (reads, writes, .observers.attach(…) and .bind(…) sites) in one undoable operation. Names starting with _ are exempt.
