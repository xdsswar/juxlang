Jux

A systems programming language with Java-readable syntax, Rust-grade safety, and native LLVM-class performance. The compiler emits Rust source that any stable rustc builds; final binaries match hand-written Rust speed.

Java syntax Rust safety Native speed unsafe { } escape No GC

Architecture at a Glance

Jux replaces rustc's top layers (lex, parse, type check, borrow check, MIR) with its own. It then emits Rust source code that any stable rustc can compile. Final binaries are bit-identical to native Rust output — same LLVM IR, same machine code, same speed.

Architecture: replace rustc's top layers with Jux's; ride on rustc's bottom layers for codegen and optimization.

Compilation Pipeline

The Jux compiler ships as a multi-phase pipeline. Each phase consumes a typed IR and produces the next. The reachability pass (9.5) is the source-level tree-shaker: only items the program can actually reach are emitted to Rust.

Phases 1–18. Reachability (9.5) prunes unused functions/types before MIR — saves rustc work, not just final binary space.

Project Structure

The compiler implementation is a Cargo workspace of focused crates. There is no separate Jux-written stdlib in Phase 1 — the standard library is the Rust std, reached through the emitted Rust crate.

Workspace layout. Each crate is small and focused. Most users only see juxc and jux.

Three Build Profiles

The same Jux source compiles to three target classes depending on the profile. Features adapt to what the target supports.

Same syntax, same borrow checker, same FFI on every profile. The profile decides what's available.

Quick Tour

What Jux looks like when you actually write it.

Procedural style — structs + free functions, no classes

package com.example.physics;

public const double G = 9.81;

public struct Particle {                       // plain data, value type
    double x; double y;
    double vx; double vy;
    double mass;
}

public Particle step(Particle p, double dt) {     // free function
    return new Particle(
        p.x + p.vx * dt, p.y + p.vy * dt,
        p.vx, p.vy - G * dt,
        p.mass
    );
}

public double kineticEnergy(Particle p) {
    return 0.5 * p.mass * (p.vx * p.vx + p.vy * p.vy);
}

// Top-level in main.jux:
var p = new Particle(0.0, 100.0, 5.0, 0.0, 1.0);
for (var t : 0..100) p = step(p, 0.01);

// Zero classes. C ergonomics + Rust safety + generics + ADTs.

Enums — C, Java, AND Rust styles, all in one construct

// Pattern A — Java-style with methods
public enum Direction {
    North, South, East, West;
    public Direction opposite() {
        return switch (this) {
            case North -> South;  case South -> North;
            case East  -> West;   case West  -> East;
        };
    }
}

Direction.values();              // [North, South, East, West] — auto-derived
Direction.North.name();          // "North"
Direction.fromName("east");       // East (case-insensitive)

// Pattern B — C-compatible for FFI
@layout(c, repr = "i32")
public enum HttpStatus {
    Ok = 200, NotFound = 404, ServerError = 500
}

// Pattern C — sum types with payloads (Rust-style ADTs)
public enum Event {
    Click(int x, int y),
    KeyPress(String key),
    Resize(int w, int h)
}

switch (event) {
    case Click(var x, var y) -> print($"click ($x, $y)");
    case KeyPress(var k)     -> print($"key $k");
    case Resize(var w, var h) -> print($"resize ${w}x${h}");
}  // exhaustive — compiler verifies

Records auto-derive everything — zero boilerplate

public record Vec3(double x, double y, double z) {
    // operator==, operator hash, operator string auto-derived
    // copy on assignment is implicit (records are value types)

    public Vec3 operator+(Vec3 o) { return new Vec3(x+o.x, y+o.y, z+o.z); }
    public double operator*(Vec3 o) { return x*o.x + y*o.y + z*o.z; }  // dot
    public int operator<=>(Vec3 o) { return magnitude() <=> o.magnitude(); }
    public double magnitude() { return sqrt(x*x + y*y + z*z); }
}

var a = new Vec3(1.0, 2.0, 3.0);
var b = new Vec3(4.0, 5.0, 6.0);

print(a == b);             // false  — auto-derived equality
print(a < b);              // true   — auto-derived from operator<=>
print($"a = $a");          // "a = Vec3(x: 1.0, y: 2.0, z: 3.0)"

C#-style properties — get / set / init / expression-bodied

public class User {
    public String name  { get; set; }                    // auto, read-write
    public String id    { get; init; }                   // init-only setter
    public int    age   { get; private set; }            // asymmetric visibility
    public String email { get; set; } = "";              // with initializer

    public String displayName => $"$name (id: $id)";          // expression-bodied

    public String password {                                 // full body with validation
        get => _hash;
        set {
            if (value.isEmpty()) throw new IllegalArgumentException();
            _hash = hash(value);
        }
    }

    private String _hash;
}

Default methods on interfaces — write once, inherit everywhere

public interface Repository<T> {
    // Abstract — every implementer MUST provide
    T? findById(int id);
    void save(T item);
    void deleteById(int id);
    Iterable<T> all();

    // Default — implementers inherit, may override
    default bool exists(int id) => findById(id) != null;

    default int count() {
        var n = 0;
        for (var _ : all()) n = n + 1;
        return n;
    }

    default T findOrThrow(int id) throws NotFoundError {
        var item = findById(id);
        if (item == null) throw new NotFoundError($"id $id");
        return item;
    }

    default Iterable<T> findWhere((T) -> bool pred) => all().filter(pred);
    default bool isEmpty => count() == 0;
}

// Implementer writes only the 4 abstract methods. Everything else
// (exists, count, findOrThrow, findWhere, isEmpty) comes for free.
public final class UserRepo implements Repository<User> {
    private Map<int, User> storage = new Map<>();

    @Override public User? findById(int id) => storage.get(id);
    @Override public void save(User u) { storage.put(u.id, u); }
    @Override public void deleteById(int id) { storage.remove(id); }
    @Override public Iterable<User> all() => storage.values();
}

var repo = new UserRepo();
repo.save(new User(1, "Alice"));
print(repo.exists(1));            // true (default impl)
print(repo.count());              // 1     (default impl)
print(repo.isEmpty);              // false (default impl, computed)

Virtual dispatch — through classes and interfaces

// Through a class hierarchy
public class Shape {                              // extendable by default
    public void render() { ... }
}
public final class Circle extends Shape {
    @Override public void render() { draw_circle(radius); }
}

public void drawAll(List<Shape> shapes) {
    for (var s : shapes) s.render();    // virtual — chosen at runtime
}

// Through an interface (abstract methods + default methods)
public interface Drawable {
    void render();                            // abstract
    default void renderTwice() {                // default — overridable
        render();  render();
    }
}

public final class Sprite implements Drawable {
    @Override public void render() { gpu.blit(tex); }
}

public void renderEach(List<Drawable> items) {
    for (var d : items) d.render();        // virtual through interface
}

// Sealed interface — exact dispatch (compiler knows all implementers)
public sealed interface Token permits Ident, Number, Punct {
    String name { get; }
}
public final record Ident(String value) implements Token {
    public String name => "ident";
}

Method overridability — Java-style default

public class Animal {                                // extendable by default
    public void greet() { print("Hello"); }       // overridable (default)
    public final void breathe() { ... }           // NOT overridable — final
    public const void heartbeat() { ... }         // NOT overridable — const (synonym)
    protected void hunt() { ... }                 // overridable (default)
    private void digest() { ... }                 // NOT inherited at all
}

public class Dog extends Animal {
    @Override public void greet() { print("Woof"); }     // OK
    @Override public void breathe() { ... }              // ERROR — final
    @Override protected void hunt() { ... }              // OK
}

// Rules (Java-style):
//   • Classes are extendable by default. Mark `final`/`const` to forbid extension.
//   • Public/protected methods are overridable unless final/const.
//   • Private methods are not inherited at all.

Type test with => (replaces instanceof)

public void describe(Animal a) {
    if (a => Dog d) {                  // "if a is a Dog, bind it as d"
        d.bark();
        d.learn("sit");
    } else if (a => Cat c) {
        c.purr();
    } else if (a => Bird) {       // test without binding
        print("it's a bird");
    }
}

Switch uses Java-14 arrow syntax — no fallthrough

public void handle(HttpStatus s) {
    switch (s) {
        case OK, CREATED, ACCEPTED -> log.info("success");     // multi-pattern
        case NOT_FOUND -> log.warn("missing");
        case SERVER_ERROR -> {                                // block body
            log.error("server failure");
            metrics.incrementErrorCount();
        }
        default -> log.warn("unhandled");
    }
}

Sealed types + pattern matching + exhaustiveness

public sealed enum HttpResult {
    Ok(int status, String body),
    Redirect(String location),
    Error(int code, String message);
}

public String handle(HttpResult r) {
    return switch (r) {
        case Ok(var s, var b) -> $"$s: $b";
        case Redirect(var url) -> $"-> $url";
        case Error(var c, var m) when c >= 500 -> $"server error: $m";
        case Error(var c, var m) -> $"client error: $m";
    };  // compiler verifies every case is covered
}

Async without Future<T> — Kotlin-shaped, TS-ergonomic

public async User fetchUser(int id) {                    // returns User, not Future<User>
    var response = await http.get($"/users/$id");
    return parseUser(await response.text());
}

public async UserPage loadPage(int id) {                  // fan out, await all
    var userTask  = spawn(() -> fetchUser(id));
    var postsTask = spawn(() -> getUserPosts(id));
    return new UserPage(await userTask, await postsTask);
}

Annotations are case-insensitive — write it any way you want

@Override public void render() { ... }       // canonical
@override public void render() { ... }       // also valid — same annotation
@OVERRIDE public void render() { ... }       // also valid — same annotation

@cfg(os = "linux") public void foo() { ... }
@CFG(OS = "linux") public void foo() { ... }   // even arg names are case-insensitive

// The compiler echoes back YOUR casing in errors and juxc fmt
// preserves it — your code reads the way you wrote it.

Casts — two equivalent forms (use whichever feels right)

// Java/C/C++/C# style — prefix
var n = (int) someLong;
var d = (Dog) someAnimal;
var p = (byte*) buffer;          // (inside unsafe only)

// Rust/Kotlin/Jux style — postfix
var n = someLong as int;
var d = someAnimal as Dog;
var p = buffer as byte*;         // (inside unsafe only)

// Identical semantics. Pick whichever reads better at the call site.

Variable declarations — Java-style, type-first

// Explicit type goes BEFORE the identifier (Java-style)
int count = 42;
String name = "Alice";
List<User> users = new List<User>();
fn(int) -> int doubler = x -> x * 2;
StackString<32> sensorId = "sensor_01";

// Or use `var` to infer the type
var count = 42;                       // inferred as int
var name = "Alice";                   // inferred as String
var users = new List<User>();        // inferred as List<User>

// NO colon-annotation form (Kotlin/TS-style is rejected)
// var x: int = 5;     ← compile error E0144
// Use either:  int x = 5;   OR   var x = 5;

Function pointers, closures, method refs — three first-class forms

// 1. Closures — first-class function values, may capture state
var doubler = (int x) -> x * 2;          // type: (int) -> int
var threshold = 100;
var aboveThreshold = (int x) -> x > threshold;   // captures `threshold`
users.map(u -> u.name);                          // passed as arg

// 2. Function pointers — raw code address, no captures, FFI-compatible
public int compare(int a, int b) { return a - b; }
fn(int, int) -> int cmp = compare;     // fn pointer to a named function
var result = cmp(5, 3);                   // safe — calls a safe fn pointer

// 3. Method references — Type::method or instance::method
var ref1 = User::greet;                 // unbound: (User) -> void
var ref2 = alice::greet;                 // bound to alice: () -> void
var ref3 = User::new;                  // constructor ref

// All three compose with generics — store in collections
List<(int) -> int> transforms = new List<>();          // list of closures
List<fn(void*) -> void> ffiCallbacks = new List<>();    // list of fn pointers
List<void*> opaqueHandles = new List<>();              // list of opaque pointers

const ≡ final — synonyms everywhere

// Bindings (locals, fields, params)
const var pi = 3.14159;
final var name = "Alice";        // identical meaning
const String greeting = "Hi!";

// Class declarations — non-extensible (no other class can extend)
public final class User { ... }
public const class User { ... }       // identical — same non-extensible semantics

// Method declarations on extendable classes — non-overridable
public final void render() { ... }
public const void render() { ... }      // identical

// Pick whichever feels natural — compiler echoes back YOUR spelling.

Address-of &, dereference * — C/C++-style inside unsafe

unsafe {
    var x: int = 42;
    var p: int* = &x;             // address-of
    *p = 100;                       // dereference + write
    print(x);                       // 100

    var arr = new int[10];
    var firstPtr = &arr[0];
    var q = firstPtr + 5;          // pointer arithmetic
    *q = 99;                        // arr[5] = 99
}

// Outside unsafe, & and * (deref) are compile errors.
// Pointers exist as a type but are opaque — pass-through only.

FFI through unsafe { } — wrapped in a safe class

@extern(lib = "sqlite3")
unsafe native {
    int sqlite3_open(CString path, out RawHandle* db);
    int sqlite3_close(RawHandle* db);
}

public final class Database {                    // safe wrapper
    private RawHandle* handle;

    public new(String path) throws DbError {
        var h: RawHandle* = null;
        var rc = unsafe { sqlite3_open(path.toCString(), out h) };
        if (rc != 0) throw new DbError("open failed");
        this.handle = h;
    }

    drop {
        // SAFETY: handle is owned by this Database; this is the unique drop
        unsafe { sqlite3_close(handle); }
    }
}

Embedded — same syntax on a 32 KB MCU

// File: blink.jux  ·  Profile: jux-core  ·  Target: thumbv7em-none-eabihf

@register(address = 0x40023830)
public static volatile uint RCC_AHB1ENR;

@register(address = 0x40020014)
public static volatile uint GPIOA_ODR;

public void main() {
    setup();
    while (true) {
        GPIOA_ODR = GPIOA_ODR ^ (1 << 5);
        delay_ms(500);
    }
}
